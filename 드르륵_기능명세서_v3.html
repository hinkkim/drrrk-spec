4. 법적 요구사항 및 컴플라이언스

이자제한법

이자제한법 및 동법 시행령에 따라 금전 대차 계약의 최고 이자율은 연 20%로 제한된다.

- 법적 근거: 이자제한법 제2조, 이자제한법 제2조제1항의 최고이자율에 관한 규정

- 월 환산: 연 20% ÷ 12 = 월 1.66%

- 시스템 적용: 이자율 슬라이더 상한을 월 1.66%로 하드 잠금 처리. 초과 입력 불가

- 표시 의무: 이자율 설정 화면에 법정 최고이자율 안내 박스 상시 노출

- 무효 조항: 법정 상한 초과 이자 약정은 초과분에 한해 무효 (민법 제103조)

⚠ ⚠ 개발 주의사항
이자율 슬라이더는 절대 비활성화하거나 우회 경로를 제공해서는 안 된다. QA 단계에서 반드시 상한값(1.66%) 초과 입력 시도를 테스트해야 한다.

4.2 대부업법

대부업자는 대부 계약 체결 전 상품 설명 의무 이행 (대부업법 제6조의4)
계약서에는 이자율·연체이자율·기간·원금을 명시해야 함
전당포 바이어는 드르륵 플랫폼 가입 시 대부업 면허 번호 및 유효기간 제출 필수
면허 만료 30일 전 자동 알림 발송, 만료 시 견적 제출 기능 잠금
유질(流質) 계약: 채무 불이행 시 담보물을 채권자가 취득할 수 있는 계약. 전당포 계약 특성상 유질 조항 포함 여부를 계약서에 명시

4.3 전자서명법

전자계약 체결 시 공인전자서명 또는 이와 동등한 수준의 인증 수단 사용(모두싸인 API)
서명 완료 계약서는 원본성 유지를 위해 변경 불가 상태로 보관
계약서 PDF 원본 해시값(SHA-256) 생성 및 블록체인 타임스탬프 기록
계약서 보관 기간: 계약 종료 후 5년 (전자문서 및 전자거래 기본법 제5조)

4.4 개인정보보호법

수집하는 개인정보: 성명, 연락처, 계좌 정보, 사업자등록번호, 대부업 면허 번호

⚠ 거래검토 단계 정책

거래검토 단계는 실물 검수, 조건 협의, 법정 증빙 확인이 이루어지는 핵심 오프라인 검토 단계로 정의한다. 이 단계에서 바이어는 유저와 직접 연락하여 담보물 상태, 대출 실행 조건, 법정 증빙 서류 확인 여부를 협의한다. 플랫폼은 거래 상태와 리마인드 UX를 제공하되, 조건 협의 자체는 플랫폼 밖 직접 연락을 통해 진행한다. 별도의 '담보가 조정 요청' 기능은 제공하지 않는다. 확인되지 않은 정보로 인한 과잉대부 책임은 대부업자 본인에게 귀속된다.

📋 전자계약 단계 정책

전자계약 단계는 거래검토 단계에서 협의가 완료된 최종 대출 조건을 바탕으로 계약을 생성하는 최종 확정 단계로 정의한다. 바이어는 `거래 확정 할게요` 버튼 클릭 후, 최종 대출 실행 조건을 다시 입력해야 하며, 해당 입력값이 전자계약서 생성의 원본 데이터가 된다. 바이어가 먼저 서명한 뒤 유저에게 계약서를 전달하며, 유저는 서명 전에 최종 조건 요약 정보를 확인하고 동의한 후 전자서명을 진행한다.

9. 용어 정의 (Glossary)

본 용어 정의는 문서 개요 참조용으로 제공됩니다. 플랫폼 내 용어와 개념의 통일된 기준으로 활용하세요.

용어
정의
대부중개업
금전의 대부 또는 대부의 중개를 영업으로 하는 업종. 직접 대출 실행 없이 중개만 수행 (대부업법 제2조)
역경매 (Reverse Auction)
판매자(전당포 바이어)가 서로 경쟁하여 구매자(유저)에게 최적 조건을 제시하는 방식. 일반 경매와 반대 방향
LTV (Loan-to-Value)
대출잔액 ÷ 담보물 감정가 × 100. 담보 대비 대출 비율. 높을수록 바이어 리스크 증가
유질물 (流質物)
채무자(유저)가 이자를 납부하지 못해 담보물에 대한 소유권이 채권자(전당포 바이어)에게 넘어간 물건
Prime Buyer
드르륵 플랫폼이 부여하는 바이어 구분. 거래 완주율·응답 속도·연체율·면허 유효성 기반 산정 (최근 90일 기준). 구분: Prime 바이어 / 일반 바이어
딜 클로징 파이프라인
낙찰 후 대출 실행까지의 4단계 프로세스: 상품발송 → 거래검토 → 전자계약 → 대출실행
DND (Do Not Disturb)

15. 리뷰 정책 세부 규정

플랫폼 신뢰도의 핵심인 리뷰 시스템의 세부 운영 규정을 정의한다. 리뷰 작성 시점, 삭제 기준, 허위 리뷰 처리 절차, 분쟁 중 리뷰 유예 정책을 포함한다.

11.1 리뷰 정책 — 1거래 1리뷰 카드 (2시점 누적)

기능명
상세 설명
1거래 1리뷰 카드
거래 1건당 리뷰 카드 1개를 생성한다. 하나의 거래 안에서 최대 2개의 시점 리뷰(1차·2차)를 누적 작성할 수 있다. 리뷰 2건은 별도 게시글로 분리하지 않고, 동일 거래 리뷰 카드 안에 타임라인 형태로 누적 노출한다.
1차 리뷰 (초기 경험)
대출 실행 완료 D+1에 유저에게 1차 리뷰 요청을 발송한다. 1차 리뷰는 입금 속도·조건 설명·응대 만족도 등 초기 경험을 평가하며, Prime 점수의 초기 공백을 줄이기 위한 데이터 수집 목적이다. 태그 예시: 입금 빨라요 / 설명 친절해요 / 검토가 빨라요
2차 리뷰 (거래 완료)
최종 상환 완료 즉시 2차 리뷰 요청을 발송한다. 2차 리뷰는 상환 과정 포함 전체 거래 경험을 평가하는 공식 평판 신호다. 2차 리뷰 작성 시 리뷰 카드에 '상환 완료 리뷰' 배지를 부여한다. 태그 예시: 상환 안내 정확해요 / 문의 응대 빨라요 / 끝까지 신뢰돼요
상환 완료 리뷰 배지
2차 리뷰가 작성된 리뷰 카드에는 '상환 완료 리뷰' 배지를 부여한다. 거래 중도 해지·취소 건은 1차 리뷰까지만 허용하고 배지를 부여하지 않는다.
리뷰 카드 타임라인 UI
리뷰 카드 상세 화면에서 [대출 실행 리뷰]와 [상환 완료 리뷰]를 시간순으로 누적 노출한다. 리스트 카드에는 대표 별점·배지·대표 문구 1줄을 노출하며, 유저 닉네임은 마스킹 처리한다.
대표 별점 노출 규칙
2차 리뷰가 있는 거래: 2차 리뷰 별점을 공식 반영, 1차 리뷰는 참고값으로 축소. 2차 리뷰가 없는 거래: 1차 리뷰를 임시 대표값으로 반영.
리뷰 집계 중복 방지
리뷰 수 통계·별점 분포·긍부정어 분석은 거래 건 기준으로 집계한다(리뷰 건수 기준 아님). 동일 거래에서 1·2차 리뷰가 모두 작성되더라도 공개 리뷰 수는 1건으로 산정한다.
Prime 점수 반영 정책
긍·부정 키워드 분석은 1·2차 리뷰를 모두 수집하되, 프로필 상단 요약과 Prime 점수 공식 반영은 2차 리뷰에 더 높은 가중치를 부여한다. 상환 완료 리뷰 비율은 Prime Buyer 내부 품질 지표로 활용한다.
리뷰 필터
바이어 프로필 리뷰 리스트에서 전체 리뷰 / 상환 완료 리뷰만 보기 필터를 제공한다. 정렬: 최신순·별점 높은순.
리뷰 리마인드
1차 리뷰 미작성 시 D+3에 1회 리마인드. 2차 리뷰 미작성 시 상환 완료 후 D+3·D+7에 리마인드를 발송한다.
리뷰 수정 정책
1차 리뷰 작성 후 2차 리뷰 요청 수신 시, 새 리뷰를 생성하지 않고 기존 리뷰 카드에 '최종 후기 추가' 방식으로 진입한다. 1차 리뷰는 삭제하지 않고 거래 타임라인의 일부로 보존한다.
리뷰 답글
바이어는 리뷰 카드 단위로 1회 답글을 작성할 수 있으며, 답글은 1차/2차 리뷰 모두에 공통 연결된다.

17. 상환 관리 — 상태 머신, 알림 설계, 위험 지표

본 섹션은 유저(채무자)와 바이어(대부업자) 간 직접 송금 구조를 전제로 한 상환 관리 시스템의 상태 머신, 알림 타임라인, 입금 확인 UX, 연체 위험 지표를 정의한다. 플랫폼(드르륵)은 실제 입금을 직접 확인할 수 없으므로, 최종 완료 판정 권한은 바이어에게 있으며 유저의 송금 신고는 보조 신호로만 사용된다.

⚠ 상환 관리 설계 원칙
설계 핵심 원칙① 유저의 "송금했어요" 클릭은 보조 신호이며 필수 절차가 아님 — 유저 미클릭이 상환 실패를 의미하지 않는다.② 바이어의 "입금 확인"만이 상환 완료를 확정하는 유일한 기준이다.③ 유저 신고만으로는 완료 처리되지 않으며, 바이어 확인 전까지 "바이어 확인 대기" 상태를 유지한다.④ 연체이자 계산 기준은 바이어 클릭 시각(confirmed_at)이 아닌 실제 입금 일시(actual_received_at)이다.⑤ 운영상 외부 연체 노출(D+1 18:00)과 내부 shadow accrual 기산(D+1 00:00)은 분리 운영한다.

11.5 Shadow Accrual 운영 원칙

⚖️ 법적 참고 조항
• 이자제한법 제2조 일할 이자: 연 20% ÷ 365 = 일 이자율 (초과 적산 시 시스템 오류 처리)
→ 섹션 11.5 상세 법적 가이드 보기

18. 대출 연장하기 (후순위 개발 — Post-MVP)

MVP에서 제외된 기능으로, 기본 대출 플로우(의뢰 → 낙찰 → 딜 클로징 → 상환) 안정화 이후 2차 스프린트에서 개발한다. 유저가 만기일에 원금을 상환하지 못할 경우 이자를 납부하고 대출 기간을 연장하는 기능이다.

11.1 개발 배경 및 우선순위

MVP 제외 사유: 연장 기능은 단순 기간 연장 UI 이상으로, 이자율 재산정
• 신규 전자계약 생성
• LTV 재검토
• 블록체인 타임스탬프 재발행 등 백엔드 복잡도가 높아 MVP 범위에서 제외하였다.

후순위 개발 필요성: 최소 대출기간 1개월 설정 시, 만기 도래 후 원금 상환이 어려운 유저의 유일한 대안이 유질 처분으로 직행되는 문제가 발생한다. 연장 기능 없이는 유저 이탈 및 플랫폼 신뢰도 저하 위험이 있다.

개발 우선순위: MVP 안정화 → 유질물 처분 플로우 고도화 → 대출 연장하기 순서. 2차 스프린트 내 포함 권장.

11.6 법적 고려사항

이자제한법 (연 20% 상한): 연장 이자율은 원 계약 이자율 이상으로 바이어가 제안하나, 연 20%(월 1.66%)를 절대 초과할 수 없다. 시스템에서 슬라이더 하드 잠금 및 서버 측 유효성 검증 이중 적용.

대부업법 이자 재산정 금지: 연장 시 기존 연체이자를 원금에 합산하여 복리 계산하는 행위 금지. 연장은 신규 기간에 대한 단리 이자만 적용.

전자서명법: 연장 계약서는 원 계약서와 동일한 SHA-256 전자서명 + 블록체인 타임스탬프 적용. 원 계약서 ID를 연장 계약서에 명시하여 계약 연속성 입증.

바이어 거절 권한: 바이어는 연장을 거절할 수 있으며, 이 경우 유저는 만기일까지 원금+이자를 상환하거나 유질 처분 플로우에 진입한다. 바이어에게 연장 강제를 불가하며, 이를 유저에게 사전 고지.

11.1 전당포 금리 (법정 최고 금리)

전당포 및 대부업 성격을 갖는 드르륵 플랫폼이 대출 이자율 설정 시 준거해야 하는 법령이다.

⚖️ 이자제한법 제2조 제1항 · 시행령 제2조
금전대차 계약의 최고이자율: 연 20% (대통령령 제34453호, 2024년 기준). 약정이자율이 연 20%를 초과하는 부분은 무효.
🔧 드르륵 플랫폼 구현
P-04 영업프로필 Step3 이자율 슬라이더 상한 월 1.66%(연 20%) 하드잠금. 서버 측 유효성 검증 이중 적용. F-04-03/F-04-04 법정 안내 박스 항상 노출.
⚖️ 대부업 등의 등록 및 금융이용자 보호에 관한 법률 (대부업법) 제8조 제1항
대부업자가 개인 또는 소기업·소상공인에게 대부하는 경우 이자율은 연 20%를 초과할 수 없음. 초과 이자를 받은 경우 형사처벌 대상.
🔧 드르륵 플랫폼 구현
바이어(대부 실행자) 이자율 입력 시 연 20% 초과 입력 서버 단 차단. 위반 시 플랫폼 이용 정지 처리 로직 포함.
⚖️ 전당포업에 관한 법률 제9조
전당업자는 이자제한법에 따른 최고이자율 이내로 이자를 징수해야 하며, 수수료·보관료 등 명목으로 이자를 우회 징수하는 행위 금지.
🔧 드르륵 플랫폼 구현
플랫폼 수수료(3%, 건별 청구) 및 이자율은 별도 항목으로 분리 명시. 수수료를 이자로 합산 계산하지 않도록 정산 로직 설계.

11.2 연체 이율

약정 기일 내 이자 미납 시 적용되는 연체 가산 이율 관련 법령이다.

⚖️ 이자제한법 제2조 제3항
연체이자율은 약정이자율의 1.5배를 초과할 수 없으며, 법정 최고이자율(연 20%)을 절대 초과하지 못함. 두 기준 중 낮은 값 적용.
🔧 드르륵 플랫폼 구현
F-06-05 연체이자 계산: 연체이자율 = Min(약정이자율 × 1.5, 연 20%). 예: 약정 월 1.2%(연 11.4%) → 연체이자율 Max 연 20% 적용. 자동 계산 로직 서버 구현.
⚖️ 대부업법 제8조 제3항
연체이자 산정 시 기존 이자를 원금에 합산하여 복리로 계산하는 행위(이자의 자본화) 금지. 단리 적용 원칙.
🔧 드르륵 플랫폼 구현
F-06-05 연체이자 계산은 단리만 적용. 연체이자 = 대출 원금 × 연체이자율 × 연체일수 / 365. 원금에 이자 합산 복리 계산 로직 구현 금지.

11.3 주의사항 — 이자·수수료 관련

이자 관련 플랫폼 운영 시 발생할 수 있는 법적 위반 유형 및 주의사항이다.

⚖️ 이자제한법 제4조 (선이자 제한)
선이자(대출 실행 전 이자 공제)를 받은 경우, 그 금액을 원금에서 차감한 금액을 실제 대출원금으로 간주. 공제 이자율 포함 시 연 20% 초과 여부 재산정 필요.
🔧 드르륵 플랫폼 구현
대출 실행 시 선이자 공제 방식 사용 금지. 대출금 전액 지급 후 약정일에 이자 수취하는 후이자 방식만 허용. 계약서 명시.
⚖️ 대부업법 제11조 (불법 수수료 금지)
대부 관련 명목으로 법에 규정되지 않은 수수료·사례금·할인료 등을 받는 행위 금지. 플랫폼 중개 수수료는 대부업법 범위 내 명시적으로 규정되어야 함.
🔧 드르륵 플랫폼 구현
플랫폼 중개 수수료 3%(건별 청구)는 계약서 및 앱 내 명시적으로 별도 항목 표시. 수수료 부과 근거를 이용약관에 명문화. 기타 명목 부가 금지.

11.4 상환방식 및 기간

11.4.1 상환방식

⚖️ 민법 제603조 (소비대차의 기간)
소비대차(금전대여)에서 당사자는 반환 시기를 자유롭게 약정할 수 있음. 기간 약정이 없는 경우 채권자는 상당한 기간을 정하여 반환 청구 가능.
🔧 드르륵 플랫폼 구현
계약서에 대출 기간(만기일)을 명시적으로 기재. U-13 만기상환 화면에 잔여일 카운트다운 표시. 기간 약정 없는 대출 계약 체결 불가.
⚖️ 대부업법 제6조 제2항 (계약서면 기재사항)
대부계약 체결 시 서면에 반드시 명시: 대부금액, 대부이자율, 변제기간 및 방법, 연체이자율. 서면 미교부 시 과태료 처분.
🔧 드르륵 플랫폼 구현
F-03-05 전자계약서: 대출금액·이자율·대출기간·만기일·상환방법·연체이자율 전항목 자동 삽입. 서명 전 유저가 주요 조항 확인하는 미리보기 단계 필수.

11.4.2 상환기간

⚖️ 전당포업에 관한 법률 제11조 (전당기간)
전당기간은 1개월 이상으로 하며, 기간 만료 전 유저(전당인)에게 기간 만료 사실을 통지해야 함. 통지 없이 유질 처분 불가.
🔧 드르륵 플랫폼 구현
최소 대출기간 1개월 설정. U-13 만기상환: D-7/D-3/D-1 앱 푸시 + 이메일 자동 발송. F-06-06 상환 일정 사전 알림 의무화.

11.5 이자 납부

⚖️ 민법 제379조 (법정이자율)
법정이자율은 연 5%. 당사자 약정이 있는 경우 약정이자율 우선 적용. 약정이자가 법정최고이자율 초과 시 초과분 무효 (채무자 초과 이자 반환 청구 가능).
🔧 드르륵 플랫폼 구현
약정이자율이 법정 한도 초과 시 자동 보정 로직 구현. 초과 납부된 이자는 원금 상환에 먼저 충당하는 처리 로직 구현.
⚖️ 대부업법 제6조 제4항 (이자 영수증 교부 의무)
대부업자는 이자를 받은 경우 이자 영수증(이자 수령 확인서)을 의무적으로 교부해야 함. 미교부 시 과태료.
🔧 드르륵 플랫폼 구현
F-06-03 이자 수령 확인 팝업: 확인 버튼 클릭 시 이자 납부 영수증 자동 생성 및 유저 앱 내 저장. PDF 다운로드 기능 제공. 영수증에 납부일·금액·잔여원금 명시.
⚖️ 이자제한법 제3조 (이자의 사전 공제 제한)
이자를 사전 공제하는 방식으로 최고이자율을 우회하는 경우 실질 이자율 기준으로 판단. 이자 계산 시 실제 사용 기간 기준으로 적용.
🔧 드르륵 플랫폼 구현
월 이자 납부 방식(후이자)으로 고정 운영. 이자율 계산은 실제 대출 사용 일수 기준. 조기 상환 시 잔여 이자 비례 반환 로직 구현.

11.6 물건 처분 (유질)

연체 장기화 시 담보물 처분 관련 법적 절차 및 제한사항이다. 이 영역은 법적 리스크가 가장 높으므로 법무 검토를 반드시 거쳐야 한다.

11.6.1 처분 시기

⚖️ 전당포업에 관한 법률 제14조 (전당물의 처분)
전당기간 만료 후에도 유저(전당인)가 원리금을 상환하지 않는 경우, 전당업자는 전당물을 처분할 수 있음. 단, 처분 전 유저에게 사전 통지 의무.
🔧 드르륵 플랫폼 구현
F-07-06 유질 처분 요건 체크리스트: 만기 경과 확인, 사전 통지 완료, 14일 유예 경과 확인. 3가지 요건 충족 시에만 즉시팔기·맡겨팔기 버튼 활성화.
⚖️ 민법 제339조 (유질계약)
민법은 유질계약(채무 불이행 시 질물로 채무를 갈음하는 계약)을 원칙적으로 금지. 단, 상행위(상법 제59조) 및 전당포업법에 따른 전당업에서는 예외 허용.
🔧 드르륵 플랫폼 구현
드르륵은 대부중개 플랫폼(상행위)으로서 상사질권 성격의 유질 처분 가능. 이용약관 및 계약서에 유질 조항 명시 필수. F-03-05 전자계약서 유질 조항 필수 기재.

11.6.2 처분 절차

⚖️ 전당포업에 관한 법률 제14조 제2항 · 제15조
처분 전 유저에게 내용증명 우편으로 처분 예정 사실 통보 의무. 통보 후 14일 이상 경과 후 처분 가능. 통보 없이 처분 시 민·형사상 책임 발생.
🔧 드르륵 플랫폼 구현
F-07-06 체크리스트 2번째 항목: 내용증명 발송 완료 확인. 3번째 항목: 발송 후 14일 경과 확인. Admin 연체 스케줄러 D+30 알림으로 처분 요건 충족 여부 자동 감지.
⚖️ 민법 제342조 (질권의 목적물에 관한 채권)

11.7.3 탭바 설계 추가 제안

아래는 UX 개선을 위한 추가 제안 사항이다.

• User OS — "알림" 탭 추가 제안: 견적 도착, 대출 만기 D-7/D-3/D-1 알림, 상환 완료 등을 별도 탭으로 분리하면 알림 중심 사용 행태에 효과적 (탭 4개→5개로 확장, 대출신청 FAB 유지)

• User OS — 탭 뱃지: 받은 견적 탭에 미확인 견적 수 뱃지 표시

• Buyer OS — "리스크 관리" 탭 검토: 진행 중 거래 탭 안에 서브탭으로 통합하거나 별도 탭으로 분리하는 방안 검토 필요 (현재 5탭 기준 유지 권장)

• 공통 — 탭바 중앙 FAB: User OS 3번 탭(대출신청)처럼 Buyer OS도 2번 탭(입찰하기)을 FAB 스타일로 강조 처리 권장
